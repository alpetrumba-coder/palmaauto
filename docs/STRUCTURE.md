# Структура проекта «ПальмаАвто»

Кратко: **вся разработка сайта ведётся в каталоге `c:\cursor\palmaauto`** (это корень приложения).

## Корень проекта (`c:\cursor\palmaauto`)

| Путь | Назначение |
|------|------------|
| `package.json` | Зависимости и скрипты (`dev`, `build`, работа с Prisma). |
| `tsconfig.json` | Настройки TypeScript и алиас импорта `@/…` → корень проекта. |
| `next.config.ts` | Настройки Next.js (сборка, пути и т.д.). |
| `eslint.config.mjs` | Правила ESLint для единообразного кода. |
| `.env` | **Локальные секреты** (не коммитим). Скопируйте из `.env.example`. |
| `.env.example` | Пример переменных окружения с пояснениями. |
| `.gitignore` | Что не попадает в Git (node_modules, .env, .next). |

## `app/` — маршруты и интерфейс (App Router)

Next.js связывает **папки** с **URL**. Скобки в имени папки — **группа маршрутов**: она **не** попадает в URL, но позволяет задать **свой layout** для зоны сайта.

| Путь | Назначение |
|------|------------|
| `app/layout.tsx` | **Корневой layout**: подключает глобальные стили, оборачивает все страницы. |
| `app/globals.css` | Дизайн-система: CSS-переменные, типографика, сетка (стиль близкий к «чистым» интерфейсам вроде Apple). |
| `app/(public)/` | **Публичная зона** (лендинг, каталог, позже — вход). Общая «оболочка» для посетителей. |
| `app/(public)/layout.tsx` | Шапка/подвал публичной части. |
| `app/(public)/page.tsx` | Главная страница `/`. |
| `app/(public)/cars/page.tsx` | Каталог `/cars` (только активные машины из БД). |
| `app/(public)/cars/[slug]/page.tsx` | Карточка авто `/cars/…`. |
| `app/(internal)/` | **Внутренняя зона** (сотрудники/админ). Позже сюда добавят защиту по роли. |
| `app/(internal)/layout.tsx` | Отдельная оболочка для внутренних экранов. |
| `app/(internal)/staff/page.tsx` | Заглушка по адресу `/staff` (второй layout без конфликта с главной `/`). |

## `components/`

Переиспользуемые **React-компоненты** (кнопки, карточки, блоки шапки). См. `components/README.md`.

## `lib/`

**Утилиты и серверный код без UI**: клиент Prisma, хелперы. См. `lib/README.md`.

## `prisma/`

| Путь | Назначение |
|------|------------|
| `schema.prisma` | Модели БД и подключение к PostgreSQL. |
| `migrations/` | История миграций — версии схемы для продакшена и команды. |

## `public/`

Статические файлы по URL: `/favicon.ico`, иконки и т.д.

**`public/cars/<slug>/`** — фотографии автомобилей для каталога: в БД хранится путь вида **`/cars/<slug>/файл.jpg`**. Памятка: **`public/cars/README.txt`**.

## `app/admin-panel/`

Отдельная зона админ-панели (cookie-сессия, не NextAuth-логин пользователя): **`/admin-panel/login`**, после входа — **`/admin-panel`**, каталог машин — **`/admin-panel/cars`** (список, создание **`/admin-panel/cars/new`**, правка **`/admin-panel/cars/[id]/edit`**). Layout с проверкой сессии: **`app/admin-panel/(dashboard)/`**.

---

Дальнейшие этапы плана (каталог, авторизация, брони) расширят `app/`, `components/` и `prisma/schema.prisma` по тем же принципам: новая фича — понятная папка и короткий комментарий в коде.
